configtxtgen

This is to generate config
Its a CLI for managing configuring Artificates
It is mostly used to generate following artificats
	- Genesis block
	- Channel Tx
	- Anchor Peer Transcation

Configtx yamlfile setup

The Configtx.yaml file has 6 section
	- Organization //List the member organization in the network
	- Orderer // Configuration for the orderers
	- Application // configuring parameters related to applications
	- Channels // configuring parameters related to applications
	- Capabilities // Binary version management accross the network
	- Profile //used for multiple config in single file
	
in the yaml file of Organization section contains

& represnts - that it can be referred to other parts of configtx yaml file
ID - represnts - a sample name refer to MSP
MSPDir - represnts - Path to the MSP dir seprately for orderer and peer
AnchorPeers: - List of one or more Anchor peer. 
			 - The anchor peerare known in the network.
				Contains - host and   - port data

In the yaml file of Orderer contains
OrdererType: - represent message communication medium such as solo, kafka, etcdraft
Addresses: - contain list of IP and port if it is kafka or etcdraft else single IP and Port for solo 
BlockTime: howlong to wait after trancation and before block is created
Batchsize: section7,v24 explained clearly

configtxgen command [--flag]
	
	Genesis Block
	   -outputBlock -inspectBlock
	Channel Tx
	   -outputCreateChannelTx -inspectChannelCreateTx
	Anchor peer Tx
	   -outputAnchorPeersUpdate -inspectChannelCreateTx

To create a genesis block follow the below steps
Step 1: Create directory and move to it
Step 2: Generate the crypto-materials as explained in orderer-peer-cryptogen creation.txt
Step 3: Then now we can create genesis block
		configtxgen -outputBlock ./<Genesis_block_name>.block -profile <Profile-name in configtx.yml> -chanelID <Any channel name>
		 /home/apptium/fabric/fabric-samples/bin/configtxgen -outputBlock ./acme-genesis.block -profile AcmeOrdererGenesis  -channelID acmeordererChannel

		To read the data from the the genesis block
		configtxgen -inspectBlock ./<Genesis_block_name>.block

Step 4:
		To create a ChannelTx use the following command
		configtxgen -outputCreateChannelTx <Any-Channel-name>.tx -profile <AcmeChannel> -channelID acmeordererChannel
		/home/apptium/fabric/fabric-samples/bin/configtxgen -outputCreateChannelTx acme-channel.tx -profile AcmeChannel -channelID acmeordererChannel

		
		To read the channel file
		configtxgen -inspectChannelCreateTx <Channel-name>
		
Step 5: To create Anchor peer update Transcation //  This file generated by this command is used by peer binary S7,v28
		/home/apptium/fabric/fabric-samples/bin/configtxgen -outputAnchorPeersUpdate <org1Anchors>.tx -profile <AcmeChannel> -chanelID acmeordererChannel -asOrg Org1
		/home/apptium/fabric/fabric-samples/bin/configtxgen -outputAnchorPeersUpdate org1Anchors.tx -profile AcmeChannel -channelID acmeordererChannel -asOrg Org1

		
		
		To view the information about orgination
		/home/apptium/fabric/fabric-samples/bin/configtxgen -printOrg org1
